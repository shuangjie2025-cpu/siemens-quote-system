# 如何发布应用 (部署指南)

为了让您的同事在中国各地都能随时随地访问此应用，我们需要将应用发布到互联网上。
由于这是一个纯静态的网页应用（基于 React + Vite），最简单且免费的方式是使用 **Vercel** 或 **Netlify** 等静态托管服务。

虽然这些服务在国外，但在中国通常也可以访问（速度尚可）。如果您需要极致的国内访问速度，则需要购买阿里云/腾讯云服务器并备案（流程较长）。

以下是推荐的快速发布方案（使用 Vercel）：

## 方法一：使用 Vercel (推荐，最简单)

1.  **准备代码**：
    *   您当前的代码已经在 `E:\西门子报价方案\web-app`。
    *   我们刚刚已经运行了 `npm run build`，生成了 `dist` 文件夹（这是最终的网页文件）。

2.  **上传发布**：
    *   访问 [Vercel 官网](https://vercel.com/) 并注册一个账号（可以使用 GitHub 账号登录）。
    *   在 Vercel 控制台点击 **"Add New..."** -> **"Project"**。
    *   **如果您的代码在 GitHub 上**：直接在列表中选择您的仓库，点击 Import，然后点击 Deploy 即可。
    *   **如果您没有 GitHub**：
        1.  在电脑上安装 Vercel CLI 工具：打开终端运行 `npm install -g vercel`。
        2.  在项目目录运行 `vercel login` 登录。
        3.  运行 `vercel` 命令，一路回车确认即可。

## 方法二：使用腾讯云/阿里云 (适合正式商用，速度最快)

1.  **购买服务**：
    *   购买一台云服务器 (ECS) 或 对象存储 (OSS/COS) + CDN。
    *   购买一个域名（如 `siemens-quote.com`）。
    *   进行 ICP 备案（通常需要 1-3 周）。

2.  **部署**：
    *   将 `dist` 文件夹中的所有文件上传到服务器的 Nginx 目录或 OSS 存储桶中。

## 临时方案：本地局域网访问 (仅限同一 WiFi)

如果您的同事和您在同一个办公室，连接同一个 WiFi：
1.  在您的电脑上运行 `npm run dev -- --host` (我已经为您运行过这个)。
2.  查看命令行显示的 `Network: http://192.168.x.x:5173/`。
3.  让同事在手机上输入这个 IP 地址即可访问。
    *   *缺点：您电脑关机或断网，他们就无法访问；离开办公室也无法访问。*
